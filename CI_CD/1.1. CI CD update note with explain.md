# 🚀 Complete GitOps Guide: GitHub Actions → Docker Hub → Private VPS (Production)

## 📖 What is CI/CD?

### CI (Continuous Integration)

Developer যখন GitHub-এ নতুন কোড Push করে, তখন স্বয়ংক্রিয়ভাবে—

- Code Checkout হয়
- Dependencies Install হয়
- Test Run হয়
- Docker Image Build হয়

যদি Test Fail করে তাহলে Pipeline এখানেই Stop হয়ে যায়।

### CD (Continuous Deployment)

যদি CI Success হয় তাহলে—

- Docker Image Docker Hub-এ Push হবে
- VPS-এ SSH Login হবে
- Latest Image Pull হবে
- Old Container Remove হবে
- New Container Run হবে

```text
    │

    │
    ├── Checkout Code
    ├── Install Node Modules
    ├── Run Tests
    ├── Build Docker Image
    ├── Push Image to Docker Hub
    └── SSH into VPS
            │
            ▼
     Pull Latest Image
            │
            ▼
     Stop Old Container
            │
            ▼
     Start New Container
```

# Dockerfile

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./

RUN npm ci --omit=dev

COPY . .

EXPOSE 3000

CMD ["npm", "start"]
```

**Why `npm ci`?**

Production-এ `npm ci` সবসময় `npm install` থেকে দ্রুত এবং deterministic।

# GitHub Actions YAML Basics

## name

```yaml
name: Node JS CI/CD Pipeline
```

Pipeline-এর নাম।

## on

```yaml
on:
  push:
    branches:
      - main
```

main branch-এ Push হলেই Pipeline Run হবে।

## jobs

```yaml
jobs:
```

Pipeline-এর সব Job এখানে লেখা হয়।

## runs-on

```yaml
runs-on: ubuntu-latest
```

Runner Operating System।

## steps

```yaml
steps:
```

একটি Job-এর ভিতরের ছোট ছোট Task।

## uses

```yaml
- uses: actions/checkout@v4
```

Marketplace Action ব্যবহার করা হচ্ছে।

Syntax:

```text
Owner/Repository@Version
```

Examples:

```text
actions/checkout@v4
docker/login-action@v3
actions/setup-node@v4
appleboy/ssh-action@v1
```

## run

```yaml
run: npm test
```

Shell Command Execute করবে।

## with

```yaml
with:
  node-version: 20
```

Action-এর Input Parameters।

## secrets

```yaml
${{ secrets.VPS_IP }}
```

GitHub Secret Access।

## github.sha

```yaml
${{ github.sha }}
```

Current Commit Hash।

## needs

```yaml
needs: test
```

আগের Job Success না হলে পরের Job Run হবে না।

# Deploy Commands

Login

```bash
echo "${{ secrets.DOCKER_PASSWORD }}" | docker login \
-u "${{ secrets.DOCKER_USERNAME }}" \
--password-stdin
```

Pull

```bash
docker pull username/node-api:latest
```

Stop

```bash
docker stop node-api || true
```

Remove

```bash
docker rm node-api || true
```

Run

```bash
docker run -d \
  --name node-api \
  -p 3000:3000 \
  --restart always \
  --env-file .env \
  username/node-api:latest
```

# Rollback Strategy

```text
test
   │
   ▼
build
   │
   ▼
deploy
```

যদি Test Fail করে—

- Build ❌
- Deploy ❌
- Old Container চলতেই থাকবে।

`github.sha` Tag ব্যবহার করলে যেকোনো পুরোনো Image-এ Rollback করা যায়।

# Best Practices

- Never commit `.env`
- Use Docker Hub Access Token
- Use `npm ci`
- Keep Docker Images Small
- Tag Images with `github.sha`
- Use GitHub Secrets
- Enable Docker Restart Policy
- Write Unit Tests
- Enable GitHub Actions Cache
- Add Health Checks
- Monitor Logs (`docker logs`)
- Use Branch Protection
