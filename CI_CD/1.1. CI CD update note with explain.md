📖 What is CI/CD?
CI (Continuous Integration)

Developer যখন GitHub-এ নতুন কোড Push করে, তখন স্বয়ংক্রিয়ভাবে

Code Checkout হয়
Dependencies Install হয়
Test Run হয়
Docker Image Build হয়

যদি Test Fail করে তাহলে Pipeline এখানেই Stop হয়ে যায়।

CD (Continuous Deployment)

যদি CI Success হয় তাহলে

Docker Image Docker Hub-এ Push হবে
VPS-এ SSH Login হবে
Latest Image Pull হবে
Old Container Remove হবে
New Container Run হবে

Step 2: Generate SSH Key
ssh-keygen -t rsa -b 4096


Note: 4048 নয়, 4096 হলো প্রচলিত ও সঠিক RSA key size।

Public Key

Private Key

Public Key

cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys


Permission

chmod 600 ~/.ssh/authorized_keys

আপনার মূল deploy.yml-এ যে দুটি ভুল ছিল
github.sha ট্যাগে টাইপো:
tags: |
  ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest
  ${{ secrets.DOCKER_USERNAME }}/node-crud-api:${{ github.sha }}

docker login কমান্ডে টাইপো:
docker login -u ${{ secrets.DOCKER_USERNAME }} -p ${{ secrets.DOCKER_PASSWORD }}


আরও নিরাপদ বিকল্প:

echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin


এভাবে নোটটি শুধু "কী করতে হবে" নয়, "কেন করতে হবে" এবং প্রতিটি GitHub Actions YAML কিওয়ার্ডের অর্থও কভার করে, ফলে এটি একটি সম্পূর্ণ প্রোডাকশন-লেভেল রেফারেন্স হিসেবে ব্যবহার করা যাবে।
