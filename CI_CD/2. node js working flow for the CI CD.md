### যদি `npm test` Fail করে?

```text
Push
  │
  ▼
Test ❌
  │
  ├── Build হবে না
  ├── Docker Image Push হবে না
  ├── Deploy হবে না
  └── পুরোনো Container চলতেই থাকবে
```

এটাই CI/CD-এর সবচেয়ে গুরুত্বপূর্ণ flow:

> **Code Push → Test → Build → Push Docker Image → Deploy → Live** 🚀
================================== Best Practice ============================
হ্যাঁ, **এটাই best practice**। Production-এ push করার আগে **লোকালেই সব verify** করা উচিত।

### Local-এ কী কী check করবে?

```bash
# 1. Install dependencies
npm ci

# 2. Run tests
npm test

# 3. Build Docker image
docker build -t node-api:test .

# 4. Run container
docker run -d -p 3000:3000 --name node-test node-api:test

# 5. Check logs
docker logs node-test

# 6. Test API
curl http://localhost:3000
# অথবা Postman/Browser দিয়ে

# 7. Stop and remove
docker stop node-test
docker rm node-test
```

### তাহলে GitHub Actions-এর কাজ কী?

GitHub Actions আবার একই জিনিসগুলো **automation** হিসেবে চালায়:

* ✅ `npm ci`
* ✅ `npm test`
* ✅ Docker image build
* ✅ Docker Hub-এ push
* ✅ VPS-এ deploy

এটা একটি **safety net**। ধরো, তুমি ভুল করে এমন কোড push করলে যেটা local-এ test করোনি, তাহলে GitHub Actions `npm test`-এ fail করবে এবং **production deploy হবে না**।

**সংক্ষেপে:**

* **Local:** দ্রুত development, debugging, Docker image verify।
* **GitHub Actions:** automated verification + build + deployment।
* **Production:** শুধুমাত্র সফলভাবে test পাস করা image-ই deploy হবে।

এটাই professional CI/CD workflow।
