এখানে GitHub Actions এবং GitOps ব্যবহার করে আপনার Node.js CRUD API-টি আপনার নিজস্ব Private VPS (যেখানে GitLab বা সরাসরি Docker চলছে) এ ডেপ্লয় করার জন্য একটি কমপ্লিট, প্রোডাকশন-রেডি নোট দেওয়া হলো। এটি সহজে বোঝার জন্য সহজ Bangla + English (Banglish) মিক্সে লেখা হয়েছে।

## Core Concept: GitActions + GitOps কি?

* CI (Continuous Integration): কোড পুশ করার সাথে সাথে অটোমেটিক টেস্ট রান করা এবং ডকার ইমেজ তৈরি (Build) করে Docker Hub বা Registry-তে পুশ করা।
* CD (Continuous Deployment / GitOps): নতুন ইমেজ পুশ হওয়ার পর, GitHub Actions আপনার VPS-এর সাথে যোগাযোগ করবে এবং পুরানো কন্টেইনার ডিলিট করে নতুন কন্টেইনার রান করবে।
* Rollback: যদি টেস্ট ফেইল করে, তবে ডেপ্লয়মেন্ট প্রসেস ওখানেই স্টপ হয়ে যাবে, ফলে প্রোডাকশনে কোনো ভাঙা কোড (Broken code) যাবে না।

## Step 2: Dockerfile তৈরি করা
আপনার Node.js অ্যাপের জন্য একটি স্ট্যান্ডার্ড Dockerfile নিচে দেওয়া হলো:

## Step 3: GitHub Actions Workflow (deploy.yml)
এই ফাইলটি .github/workflows/deploy.yml পাথে সেভ করবেন। এটিই আপনার পুরো CI/CD পাইপলাইন কন্ট্রোল করবে।

name: Node JS CI/CD Pipeline
# শুধুমাত্র 'main' বা 'production' ব্রাঞ্চে পুশ করলে এই পাইপলাইন চলবেon:
  push:
    branches:
      - main
jobs:
  # ---- JOB 1: TEST THE APPLICATION ----
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-level: '20'

      - name: Install Dependencies
        run: npm install

      - name: Run Unit Tests
        run: npm test # এখানে আপনার 'npm test' স্ক্রিপ্ট রান হবে। টেস্ট ফেইল করলে এখানেই পাইপলাইন স্টপ (Rollback logic)!

  # ---- JOB 2: BUILD AND PUSH DOCKER IMAGE ----
  build-and-push:
    needs: test # এই জবটি চলার আগে অবশ্যই 'test' জবটি পাস হতে হবে
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Log in to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      - name: Build and Push Docker Image
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest
            ${{ secrets.DOCKER_USERNAME }}/node-crud-api:${{ github.sha }}
  # ---- JOB 3: DEPLOY TO PRIVATE VPS VIA SSH ----
  deploy:
    needs: build-and-push # ইমেজ পুশ হওয়ার পর ডেপ্লয়মেন্ট শুরু হবে
    runs-on: ubuntu-latest
    steps:
      - name: Deploying to Private VPS via SSH
        uses: appleboy/ssh-action@v1.0.3
        with:
          host: ${{ secrets.VPS_IP }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.VPS_SSH_KEY }}
          port: 22
          script: |
            # 1. লগইন টু ডকার হাব (আপনার VPS এ)
            docker login -u ${{ secrets.DOCKER_USERNAME }} -p ${{ secrets.DOCKER_PASSWORD }}
            
            # 2. লেটেস্ট ইমেজ পুল করা
            docker pull ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest
            
            # 3. পুরানো কন্টেইনার স্টপ এবং রিমুভ করা (যদি থাকে)
            docker stop node-api-prod || true
            docker rm node-api-prod || true
            
            # 4. নতুন কントেইনার রান করা
            docker run -d --name node-api-prod -p 3000:3000 --restart always ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest

## Step 4: GitHub Secrets Setup (Most Important)
আপনার পাসওয়ার্ড বা VPS IP সরাসরি কোডে লেখা সিকিউর না। তাই GitHub-এর Secrets ব্যবহার করতে হবে।

   1. আপনার GitHub Repository-তে যান।
   2. Settings -> Secrets and variables -> Actions-এ ক্লিক করুন।
   3. New repository secret বাটনে ক্লিক করে নিচের সিক্রেটগুলো যোগ করুন:


* DOCKER_USERNAME: আপনার Docker Hub-এর ইউজারনেম।
* DOCKER_PASSWORD: আপনার Docker Hub-এর পাসওয়ার্ড বা Access Token।
* VPS_IP: আপনার Private VPS-এর Public IP এড্রেস।
* VPS_USER: আপনার VPS-এর ইউজারনেম (যেমন: root বা ubuntu)।
* VPS_SSH_KEY: আপনার ল্যাপটপ/GitHub থেকে VPS-এ কানেক্ট করার SSH Private Key (যা ~/.ssh/id_rsa ফাইলে থাকে)।

## How Automated Rollback Works Here?
এই পাইপলাইনে রোলব্যাক মেকানিজমটি একদম সিম্পল এবং ফুলপ্রুফ:

   1. Pipeline Order: প্রথমে test জবটি চলে। যদি আপনার কোডে কোনো বাগ (bug) থাকে এবং টেস্ট কেস ফেইল করে, তাহলে পাইপলাইনটি "Failed" স্ট্যাটাস দেখিয়ে সাথে সাথে বন্ধ হয়ে যাবে।
   2. Next Steps Blocked: যেহেতু build-and-push এবং deploy জব দুটি needs: test এর উপর নির্ভরশীল, তাই টেস্ট ফেইল করলে তারা রানই হবে না।
   3. Result: আপনার VPS-এ আগের যে ভালো কন্টেইনারটি চলছিল, সেটিই চলতে থাকবে। প্রোডাকশনে কোনো ডাউনটাইম বা ভাঙা কোড যাবে না।
