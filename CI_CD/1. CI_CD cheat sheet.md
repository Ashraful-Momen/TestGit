এই নোটের মূল সমস্যা হলো **Markdown formatting ভেঙে গেছে**। Headings, code block, folder tree এবং YAML সবই plain text হিসেবে লেখা, তাই GitHub-এ ঠিকমতো render হবে না। নিচের পরিবর্তনগুলো করলে এটি GitHub-এর জন্য একদম clean ও production-ready হবে।

### ১. Folder Structure-কে `text` code block-এ রাখুন

````md
## Step 1: Project Structure

আপনার প্রোজেক্টের রুট ডিরেক্টরিতে এই ফাইলগুলো থাকতে হবে।

```text
my-node-app/
├── .github/
│   └── workflows/
│       └── deploy.yml
├── Dockerfile
├── .dockerignore
├── .env
├── package.json
├── package-lock.json
├── server.js
└── test/
    └── app.test.js
```
````

### ২. Dockerfile-কে `dockerfile` code block-এ রাখুন

এখন আপনার নোটে সব লাইন একসাথে লেগে গেছে:

```text
# Step 1: Base ImageFROM node:20-alpine
```

এটি ভুল।

সঠিক হবে—

````md
## Step 2: Dockerfile

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./

RUN npm ci --omit=dev

COPY . .

EXPOSE 3000

CMD ["npm", "start"]
```

### কেন `npm ci`?

Production-এর জন্য `npm ci` ব্যবহার করা ভালো কারণ এটি:

- Faster
- Deterministic
- `package-lock.json` অনুযায়ী install করে
````

### ৩. GitHub Actions YAML-কে `yaml` code block-এ রাখুন

````md
## Step 3: GitHub Actions Workflow

`.github/workflows/deploy.yml`

```yaml
name: Node JS CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:

  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Dependencies
        run: npm ci

      - name: Run Unit Tests
        run: npm test

  build-and-push:
    needs: test

    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      - uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: |
            ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest
            ${{ secrets.DOCKER_USERNAME }}/node-crud-api:${{ github.sha }}

  deploy:
    needs: build-and-push

    runs-on: ubuntu-latest

    steps:
      - uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_IP }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.VPS_SSH_KEY }}

          script: |
            echo "${{ secrets.DOCKER_PASSWORD }}" | docker login \
              -u "${{ secrets.DOCKER_USERNAME }}" \
              --password-stdin

            docker pull ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest

            docker stop node-api-prod || true
            docker rm node-api-prod || true

            docker run -d \
              --name node-api-prod \
              -p 3000:3000 \
              --restart always \
              --env-file .env \
              ${{ secrets.DOCKER_USERNAME }}/node-crud-api:latest
```
````

### ৪. GitHub Secrets-কে List বানান

```md
## Step 4: GitHub Secrets

Repository → Settings → Secrets and variables → Actions

নিচের Secret গুলো যোগ করুন:

- `DOCKER_USERNAME`
- `DOCKER_PASSWORD` *(Docker Hub Access Token ব্যবহার করা উত্তম)*
- `VPS_IP`
- `VPS_USER`
- `VPS_SSH_KEY`
```

### ৫. Rollback অংশ List আকারে লিখুন

````md
## Rollback Strategy

Pipeline Order:

```text
test
   │
   ▼
build-and-push
   │
   ▼
deploy
````

যদি `npm test` Fail করে—

* Build হবে না
* Deploy হবে না
* পুরোনো Container চলতেই থাকবে

এটাই Basic Rollback।

আরও উন্নত Rollback-এর জন্য `github.sha` tag ব্যবহার করে আগের সফল Image Deploy করা যায়।

`````

### ৬. শেষে Best Practices যোগ করুন

````md
## Best Practices

- Never commit `.env`
- Use Docker Hub Access Token
- Use `npm ci`
- Use `.dockerignore`
- Tag images with `${{ github.sha }}`
- Use GitHub Secrets
- Enable Docker Restart Policy
- Add Docker Health Check
- Enable GitHub Actions Cache
- Protect the `main` branch
- Monitor logs using `docker logs`
`````

## আরও কিছু ভুল যা ঠিক করা উচিত

* ✅ `node-level` → **`node-version`**
* ✅ `npm install --production` → **`npm ci --omit=dev`**
* ✅ `docker login -p` → **`--password-stdin`**
* ✅ `.dockerignore` যোগ করুন।
* ✅ `.env` ব্যবহারের নির্দেশনা যোগ করুন।
* ✅ `Health Check` যোগ করুন।
* ✅ `actions/setup-node@v4`, `actions/checkout@v4`, `needs`, `runs-on`, `uses`, `with`, `run`—প্রতিটির সংক্ষিপ্ত ব্যাখ্যা যোগ করুন।
* ✅ `docker/build-push-action@v6` ব্যবহার করতে পারেন (বর্তমান সংস্করণ অনুযায়ী)।
* ✅ `Self-hosted Runner` বনাম `SSH Deployment`-এর পার্থক্য নিয়ে একটি ছোট সেকশন যোগ করলে নোটটি আরও সম্পূর্ণ হবে।

এভাবে সাজালে GitHub-এ `.md` ফাইলটি সুন্দরভাবে render হবে এবং এটি একটি production-level reference হিসেবে ব্যবহার করা যাবে।
