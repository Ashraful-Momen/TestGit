**Option 1: Using Personal Access Token (Recommended for Temporary Use)**

**Step 1: Generate Personal Access Token on Your Main PC**

1. Go to GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. Click **Generate new token**
3. Configure token details:
   - **Token name**: Give it a descriptive name like "Temp CI/CD Token - PC2"
   - **Expiration**: Set to a short duration (e.g., 7 days) for security
   - **Scopes**: Select required permissions:
     - `repo` (full control of private repositories)
     - `workflow` (for CI/CD pipelines)
4. Click **Generate token** and **copy it immediately** (you'll only see it once!)
5. Store it temporarily in a secure location (never commit it to git)

---

**Step 2: Set Up Token on Other PC**

Open terminal/command prompt on the other PC and configure Git:

```bash
# Set up Git to use the token
git config --global user.name "Your Name"
git config --global user.email "your-email@gmail.com"

# Store the token securely (choose one method below)
```

**Option A: Store in Git Credential Manager (Recommended)**

```bash
# Windows
git config --global credential.helper manager-core

# Mac
git config --global credential.helper osxkeychain

# Linux
git config --global credential.helper store
```

**Option B: Store in .git-credentials file (Simple)**

```bash
# Create credentials file
echo "https://yourusername:YOUR_TOKEN@github.com" >> ~/.git-credentials

# Set permissions
chmod 600 ~/.git-credentials
```

---

**Step 3: Test Token & Start Working**

```bash
# Clone your repository
git clone https://github.com/yourusername/repo.git
cd repo

# The token should be used automatically
# Start your CI/CD work
```

**Verify token is working:**
```bash
git push origin main
# If successful, your token is properly configured
```

---

**Step 4: After Completing Your Task - Clean Up**

**Remove token from Git Credential Manager:**

```bash
# Windows
git credential-manager erase https://github.com

# Mac
git credential-osxkeychain erase https://github.com

# Linux
git credential-cache exit
```

**Remove credentials file (if you used Option B):**

```bash
rm ~/.git-credentials
```

**Clear Git configuration (optional):**

```bash
# Remove stored credentials
git config --global --unset credential.helper

# Or completely reset Git config
git config --global --list  # View current config
```

---

**Step 5: Revoke Token on GitHub (Final Step)**

1. Go back to GitHub on your main PC
2. Navigate to **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
3. Find your token (e.g., "Temp CI/CD Token - PC2")
4. Click **Delete** to revoke it immediately
5. The token becomes completely unusable now

---

**Logout from GitHub on Other PC:**

```bash
# Clear all GitHub credentials
git config --global --unset user.name
git config --global --unset user.email

# Optional: Completely clear Git cache
git credential-cache flush
```

---

**Security Checklist - After You're Done:**

✅ Token deleted from GitHub  
✅ Credentials removed from credential manager  
✅ `~/.git-credentials` file deleted (if exists)  
✅ Git config cleared  
✅ Any cloned repos deleted from the other PC  
✅ Browser history cleared (if you logged in online)  

---

**Quick Reference - Complete Workflow:**

```bash
# ON OTHER PC - SETUP
git config --global credential.helper manager-core
git clone https://github.com/yourusername/repo.git
cd repo

# DO YOUR WORK
git add .
git commit -m "Your changes"
git push origin main

# ON OTHER PC - CLEANUP
git credential-manager erase https://github.com
git config --global --unset user.name
git config --global --unset user.email
rm -rf ~/repo  # Delete cloned repository

# ON YOUR MAIN PC - FINAL CLEANUP
# Go to GitHub Settings → Delete the token
```
